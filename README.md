# MYC-MAX MD Analysis Pipeline

This project contains a the Python scripts used for analyzing Molecular Dynamics simulations of the MYC-MAX complex and its individual components.

## Results Directory Structure
All scripts automatically output to the `Results/` directory:
- `Results/Complex/`: Interface Contacts, Salt Bridges and MMPBSA for the MYC:MAX dimer.
- `Results/Individual/`: RMSD, RMSF, and Rg for single-protein replicas (MYC).
- `Results/Comparison/`: Comparative analysis (e.g., MYC alone vs. MYC in complex).
- `Results/Pockets/`: MDpocket descriptors, sub-pocket clustering, and cross-validation tables.

---

## Workflow

To ensure all dependencies are met, run the scripts in the following order:

### 1. `individual_analysis.py`
- **Purpose:** Primary analysis of individual protein replicas (e.g., MYC alone).
- **Prerequisites:** Needs `.psf` and `.xtc` files for the single-protein replicas.
- **Outputs:** Saves metrics to `Results/Individual/`.

### 2. `MYC_(MYC:MAX)_analysis.py`
- **Purpose:** Compare MYC with and without MAX.
- **Prerequisites:** Needs `.psf` and `.xtc` files for the replicas.
- **Outputs:** Saves trajectory metrics and MMPBSA results to `Results/Comparaison/`.

### 3. `general_analysis.py`
- **Purpose:** Primary analysis of the complex replicas.
- **Prerequisites:** Needs `.psf` and `.xtc` files for the complex replicas.
- **Outputs:** Saves trajectory metrics and MMPBSA results to `Results/Complex/`.

### 4. `analyze_mdpocket.py`
- **Purpose:** Analyzes cavity volume and residue composition over time.
- **Prerequisites:** Requires `_descriptors.txt` files generated by the external **mdpocket** tool.
- **Outputs:** Saves publication-quality plots and summary tables to `Results/Pockets/`.

### 5. `cross_validation.py` (Master Validation)
- **Purpose:** Integrates results from Salt Bridges, MM-GBSA, and MDpocket into a single consensus.
- **Prerequisites:** 
    - Requires `.dx` and `_atoms.pdb` files from **mdpocket**.
    - Uses results generated in Step 1 (MMPBSA/Salt Bridges).
- **Outputs:** Generates the final consensus table and a unified PyMOL script in `Results/Pockets/`.

---

## 🛠 Prerequisites
- **Python 3.8+**
- **Libraries:** `MDAnalysis`, `numpy`, `pandas`, `matplotlib`, `seaborn`, `scipy`, `gridData` (optional for sub-pocket clustering).
- **External Tools:** `gmx_MMPBSA` (for free energy), `mdpocket` (for cavity analysis).

---

## Data Handling & GitHub
Due to the significant size of Molecular Dynamics trajectories (individual `.xtc` files range from 100MB to 7GB; total project >50GB), raw data files are **excluded** from this repository. 

**To use this repository with your own data:**
1. Maintain the directory structure (e.g., `MYC-MAX/MYC-MAX-500/...`).
2. Ensure your trajectories are named according to the definitions in the `CONFIG` section of the scripts.
3. The scripts use relative path routing (via `PROJECT_ROOT`), so they will work immediately as long as the internal folder hierarchy is preserved.
